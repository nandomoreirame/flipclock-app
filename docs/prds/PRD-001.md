# PRD-001: Multi-Monitor Support for Screensaver Mode

> **Status:** approved
> **Author:** Claude Code (SDLC Workflow)
> **Created:** 2026-02-20
> **Last Updated:** 2026-02-20

---

## Overview

### Problem Statement

O FlipClock possui um modo screensaver (`--screensaver`) que atualmente abre apenas em um único monitor. Usuários com configurações multi-monitor (2+ telas) esperam que o screensaver cubra todos os monitores simultaneamente, como é o comportamento padrão de screensavers do sistema.

**Situação atual:**
- Usuário executa `./flipclock --screensaver`
- Aplicativo abre em apenas 1 monitor
- Outros monitores permanecem sem cobertura

**Configuração do usuário:**
- 2 monitores externos
- 1 tela de notebook
- Total: 3 displays

### Goal

Implementar detecção automática de múltiplos monitores e abrir uma janela fullscreen do FlipClock em cada monitor detectado quando o modo screensaver for ativado.

### Success Metrics

- [x] Detectar todos os monitores conectados (3 no caso do usuário)
- [x] Abrir uma janela fullscreen em cada monitor
- [x] Manter sincronização de horário entre todas as janelas
- [x] Funcionar em Linux (X11/Wayland) e Windows

---

## Background

### Context

O FlipClock já possui modo screensaver implementado (via `screensaver-mode.plan.md`), mas falta suporte multi-monitor. Screensavers tradicionais (XScreenSaver, Windows Screensaver) cobrem todos os monitores por padrão - usuários esperam o mesmo comportamento.

### Current State

**Implementação existente:**
- `screensaver_test.go` - testes para parsing de argumentos CLI
- `xscreensaver/flipclock.xml` - configuração XScreenSaver
- `screensaver-mode.plan.md` - plano original (single-monitor)

**Limitação técnica:**
- Fyne v2.5.1 não expõe API pública para enumeração de monitores
- Fyne usa GLFW internamente, que **suporta** multi-monitor via `glfwGetMonitors()`

### Research & Insights

**Fontes de pesquisa:**
1. [Fyne Desktop Screens API](https://pkg.go.dev/fyne.io/fynedesk) - Fornece `ScreenList` mas é específico do FyneDesk
2. [GLFW Monitor Guide](https://www.glfw.org/docs/3.3/monitor_guide.html) - `glfwGetMonitors()` retorna todos os monitores
3. [go-gl/glfw GetMonitors](https://golang.hotexamples.com/examples/github.com.go-gl.glfw.v3.1.glfw/-/GetMonitors/golang-getmonitors-function-examples.html) - Exemplos práticos em Go

**Abordagem técnica:**
- Usar `github.com/go-gl/glfw/v3.3/glfw` diretamente para detectar monitores
- Criar uma janela Fyne para cada monitor detectado
- Sincronizar atualização de tempo entre todas as janelas

## Requirements

### Functional Requirements

| ID | Requirement | Priority | Notes |
|----|-------------|----------|-------|
| FR-01 | Detectar todos os monitores conectados ao iniciar no modo screensaver | Must Have | Via `glfw.GetMonitors()` |
| FR-02 | Criar uma janela fullscreen em cada monitor detectado | Must Have | Uma instância `fyne.Window` por monitor |
| FR-03 | Exibir o mesmo horário sincronizado em todas as janelas | Must Have | Ticker compartilhado atualizando todos os clocks |
| FR-04 | Suportar 1 a N monitores (graceful degradation) | Must Have | Se detecção falhar, usar comportamento single-monitor |
| FR-05 | Posicionar cada janela corretamente no monitor correspondente | Must Have | Usar `Monitor.GetPos()` e `Monitor.GetVideoMode()` |
| FR-06 | Sair de todas as janelas ao detectar input (teclado/mouse) | Must Have | Event handler compartilhado |

### Non-Functional Requirements

| ID | Requirement | Priority | Notes |
|----|-------------|----------|-------|
| NFR-01 | Tempo de inicialização < 2s mesmo com 3+ monitores | Must Have | Criar janelas em paralelo |
| NFR-02 | Sincronização de horário com precisão de 1s entre janelas | Must Have | Mesmo `time.Now()` para todas |
| NFR-03 | Suporte a diferentes resoluções e DPIs por monitor | Must Have | Fyne handle auto-scaling |
| NFR-04 | Compatibilidade com X11, Wayland e Windows | Should Have | Testar em múltiplas plataformas |
| NFR-05 | Zero vazamento de memória em execução prolongada | Must Have | Cleanup correto de recursos GLFW |

## User Experience

### Target Users

1. **Usuário desktop multi-monitor:** Desenvolvedor/profissional com 2-3 monitores que usa FlipClock como screensaver
2. **Usuário notebook + monitor externo:** Trabalho híbrido (escritório/casa) com dock station

### User Flow

```
Usuário executa `./flipclock --screensaver`
  → App detecta 3 monitores conectados
  → App cria 3 janelas fullscreen (uma por monitor)
  → Todas exibem o mesmo horário sincronizado
  → Usuário move o mouse OU pressiona tecla
  → Todas as janelas fecham simultaneamente
```

## Scope

### In Scope

- Detecção automática de monitores via GLFW
- Criação de múltiplas janelas Fyne (uma por monitor)
- Posicionamento correto de cada janela em seu monitor
- Sincronização de horário entre janelas
- Exit unificado (qualquer input fecha todas as janelas)
- Testes unitários para detecção de monitores
- Documentação README atualizada

### Out of Scope

- Preview mode para Windows Screensaver (limitação do Fyne - já documentado)
- Configuração manual de monitores (sempre auto-detect)
- Suporte a Wayland multi-GPU (edge case - documentar limitação)
- Hot-plug de monitores durante execução do screensaver

### Assumptions

- GLFW está inicializado e funcional (Fyne já faz isso)
- Usuário tem drivers gráficos OpenGL funcionais
- Todos os monitores estão conectados antes de iniciar o screensaver
- Monitores têm geometria válida (width > 0, height > 0)

### Constraints

- Deve usar Fyne v2.5.1 (já no go.mod - não upgrade)
- Deve funcionar com Go 1.22+ (requisito do projeto)
- Não pode depender de packages externos além de `go-gl/glfw`
- Modo normal (não-screensaver) não deve ser afetado

## Risks & Mitigation

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| GLFW não inicializado quando chamamos `GetMonitors()` | Medium | High | Chamar após `app.New()` mas antes de criar janelas |
| Fyne e GLFW competem pelo contexto OpenGL | Low | High | Usar GLFW apenas para query (não criar janelas GLFW) |
| Wayland não suporta posicionamento manual de janelas | High | Medium | Documentar limitação + fallback para single-window |
| Monitores com DPI diferentes causam layout quebrado | Low | Low | Fyne já handle auto-scaling |
| Vazamento de memória com múltiplas janelas | Low | Medium | Testar com `pprof` e verificar cleanup |
