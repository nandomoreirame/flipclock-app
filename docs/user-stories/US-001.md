# US-001: Multi-Monitor Screensaver Coverage

> **Status:** ready
> **Priority:** high
> **Story Points:** 5
> **PRD:** [PRD-001](../prds/PRD-001.md)
> **Created:** 2026-02-20

---

## User Story

**As a** desktop user with multiple monitors,
**I want** the FlipClock screensaver to cover all my screens simultaneously,
**So that** I have a unified screensaver experience across my entire workspace without any monitors remaining uncovered.

---

## Acceptance Criteria

### Scenario 1: Triple Monitor Setup (Primary Use Case)

**Given** I have 3 monitors connected (2 external + 1 laptop screen)
**When** I execute `./flipclock --screensaver`
**Then** I should see:
- 3 fullscreen FlipClock windows (one per monitor)
- All windows displaying the same synchronized time
- Each window positioned correctly on its designated monitor
- All windows launching within 2 seconds

### Scenario 2: Dual Monitor Setup

**Given** I have 2 monitors connected
**When** I execute `./flipclock --screensaver`
**Then** I should see:
- 2 fullscreen FlipClock windows
- Both windows showing identical time
- Windows covering both monitors completely

### Scenario 3: Single Monitor Fallback

**Given** I have only 1 monitor connected
**When** I execute `./flipclock --screensaver`
**Then** I should see:
- 1 fullscreen FlipClock window (backward compatible behavior)
- No errors or warnings in console

### Scenario 4: Unified Exit on Input

**Given** FlipClock is running in screensaver mode on 3 monitors
**When** I press any keyboard key OR move the mouse
**Then** all 3 windows should close simultaneously within 100ms

### Scenario 5: Monitor Detection Failure Graceful Degradation

**Given** GLFW monitor detection fails for any reason
**When** I execute `./flipclock --screensaver`
**Then** the app should:
- Fall back to single-monitor mode
- Display error message in console (non-fatal)
- Launch normally on primary monitor

### Scenario 6: Mixed Resolution Monitors

**Given** I have monitors with different resolutions (1920x1080, 2560x1440, 1920x1080)
**When** FlipClock opens on all monitors
**Then** each window should:
- Adapt to its monitor's native resolution
- Maintain correct aspect ratio (5:6 for flip cards)
- Apply proper DPI scaling automatically

## Technical Notes

### Affected Components

- `main.go` - Add `detectMonitors()`, `createScreensaverWindow()`, multi-window loop
- `go.mod` - Ensure `go-gl/glfw` is available (already transitive dependency)
- New file: `monitors.go` - Monitor detection logic (optional refactor)
- New file: `monitors_test.go` - Unit tests for monitor detection

### API Changes

**New functions:**
```go
// detectMonitors returns all connected monitors or nil slice if detection fails
func detectMonitors() []*glfw.Monitor

// createScreensaverWindow creates a fullscreen window positioned on specific monitor
func createScreensaverWindow(app fyne.App, monitor *glfw.Monitor, clock *Clock) fyne.Window
```

**Modified behavior:**
- `main()` now creates multiple windows in screensaver mode instead of one

### Dependencies

- `github.com/go-gl/glfw/v3.3/glfw` - Already available as Fyne transitive dependency
- No new external dependencies required

---

## UI/UX Notes

**Visual behavior:**
- All windows show identical flip animation timing
- Cursor hidden on all monitors
- No system tray icon in screensaver mode
- Consistent color theme across all windows

**Performance:**
- Windows created sequentially (not parallel) to avoid GLFW context issues
- Each window has its own goroutine for rendering
- Shared ticker (1 second interval) updates all Clock instances

---

## Security Considerations

- Input handlers must be properly registered on ALL windows to prevent bypass
- No input should be allowed to reach underlying desktop while screensaver active
- Exit mechanism must be instant (< 100ms) for security compliance

### Unit Tests

- `TestDetectMonitors_Success` - Mock 3 monitors, verify count and properties
- `TestDetectMonitors_GlfwInitFailure` - GLFW init fails, returns empty slice
- `TestDetectMonitors_NoMonitors` - GLFW returns empty list, graceful handling
- `TestCreateScreensaverWindow_Positioning` - Verify window positioned correctly

### Integration Tests

- `TestMultiMonitorScreensaver_ThreeMonitors` - Full flow with 3 mock monitors
